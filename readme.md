# Flask メモアプリ

Udemy講座「[設計を理解しながらWebアプリを開発！【はじめてのWeb開発】【Flask】][1]」で作成したメモアプリを基に、機能追加と改善を行ったWebアプリケーションです。

ログインしたユーザーごとにメモとカテゴリを管理でき、メモの検索、絞り込み、並び替え、ページネーションなどに対応しています。

本アプリケーションはUdemy講座の教材を基に改良したものであり、元教材に関する著作権その他の権利は、教材の作成者および各権利者に帰属します。




## 開発目的

Flaskを使用したWebアプリケーション開発の学習成果として、講座で作成したアプリに独自の機能を追加しました。

主に、次の内容を目的として開発しています。

* Flaskによるルーティングと画面遷移の理解
* SQLAlchemyを使用したデータベース操作
* Flask-Loginを使用したログイン管理
* ユーザーごとのデータ分離とアクセス制御
* 検索、絞り込み、並び替え、ページネーションの実装
* Bootstrapを使用した画面デザインの改善

## 主な機能

### ユーザー管理

* ユーザー登録
* ログイン、ログアウト
* パスワードのハッシュ化
* ユーザーIDの重複チェック
* ユーザーID、パスワードの空白入力チェック

### メモ管理

* メモの新規登録
* メモの一覧表示
* メモの編集
* メモの削除
* 作成日時、更新日時の記録と表示
* タイトル、本文の空白入力チェック

### ユーザーごとのデータ管理

* ログイン中のユーザーが作成したメモのみ表示
* メモの登録者情報をデータベースに保存
* 他のユーザーが所有するメモの編集、削除を防止
* 他のユーザーが所有するカテゴリの使用を防止

### 検索・表示機能

* タイトル、本文を対象としたキーワード検索
* カテゴリによる絞り込み
* 更新日時、作成日時による並び替え
* 1ページ10件のページネーション
* 検索、カテゴリ、並び替え条件を維持したページ移動
* 不正なページ番号、カテゴリID、ソートIDへの対応

### カテゴリ管理

* ユーザーごとのカテゴリ管理
* ユーザー登録時に初期カテゴリを自動作成
* カテゴリの追加
* カテゴリ名の編集
* カテゴリの削除
* カテゴリ名の重複チェック
* 保護対象カテゴリの編集、削除防止
* 削除したカテゴリを使用していたメモを「未分類」へ移動

初期状態では、次のカテゴリが登録されます。

* 未分類
* 学習
* 仕事
* アイデア
* その他

### UI

* Bootstrap 5を使用した画面デザイン
* Flashメッセージによる処理結果の表示
* 検索、カテゴリ選択、並び替えを一覧画面に集約
* ページ番号を表示するページネーション

## データベース構成

本アプリケーションでは、主に次の3つのテーブルを使用します。

### `user`

ユーザー情報を管理します。

* ユーザー番号
* ユーザーID
* ハッシュ化されたパスワード

### `memo`

メモ情報を管理します。

* タイトル
* 本文
* 登録ユーザー
* カテゴリ
* 作成日時
* 更新日時

### `category`

ユーザーごとのカテゴリを管理します。

* カテゴリ名
* 表示優先度
* 登録ユーザー

## セットアップ方法

### 1. リポジトリを取得

```bash
git clone https://github.com/SoukenProg/Udemy_MemoApp_Improvement.git
cd Udemy_MemoApp_Improvement
```

### 2. 仮想環境を作成

#### Windows

```powershell
py -m venv .venv
```

#### macOS / Linux

```bash
python3 -m venv .venv
```

### 3. 仮想環境を有効化

#### Windows PowerShell

```powershell
.venv\Scripts\Activate.ps1
```

#### Windows コマンドプロンプト

```cmd
.venv\Scripts\activate
```

#### macOS / Linux

```bash
source .venv/bin/activate
```

### 4. 必要なパッケージをインストール

```bash
pip install -r requirements.txt
```

### 5. アプリケーションを起動

```bash
flask run --debug
```

ブラウザで次のURLを開きます。

```text
http://127.0.0.1:5000
```

データベースは、初回起動時に`instance/flaskmemo.db`として自動作成されます。

## 基本的な使い方

1. ユーザー登録画面からユーザーを登録します。
2. 登録したユーザーIDとパスワードでログインします。
3. 「新規登録」からメモを作成します。
4. 一覧画面から検索、カテゴリ絞り込み、並び替えを行います。
5. メモごとのボタンから編集または削除します。
6. 「カテゴリを見る」からカテゴリを追加、編集、削除します。

## モックデータ作成コマンド

ページネーションや検索機能の動作確認用として、指定ユーザーにモックメモを登録できます。

```bash
flask --app app seed-memos --userid ユーザーID --count 100
```

例：

```bash
flask --app app seed-memos --userid testuser --count 100
```

登録されたモックメモのタイトルには、`[MOCK]`が付与されます。

## モックデータ削除コマンド

指定ユーザーのモックメモのみ削除できます。

```bash
flask --app app delete-mock-memos --userid ユーザーID
```

例：

```bash
flask --app app delete-mock-memos --userid testuser
```

通常のメモは削除されません。

## 実装上の工夫

### ユーザー単位のアクセス制御

メモやカテゴリを取得するときは、IDだけでなくログインユーザーのユーザー番号も検索条件に含めています。

これにより、URLのIDを直接変更した場合でも、他のユーザーが所有するデータを編集または削除できないようにしています。

### カテゴリ削除時のデータ保護

使用中のカテゴリを削除した場合、関連するメモ自体は削除せず、カテゴリを「未分類」へ変更してからカテゴリを削除します。

### 検索条件を維持したページネーション

ページを移動しても、次の条件が維持されるように実装しています。

* 検索キーワード
* 選択中のカテゴリ
* 並び替え条件

### トランザクション処理

複数のデータをまとめて変更する処理では、処理に失敗した場合に`rollback()`を行い、不完全な状態でデータが保存されることを防いでいます。
